# Feature Registry

Use this file before starting a feature, follow-up, or bugfix. It is a public
duplicate-work guard, not a full implementation record.

Read order for new work:

1. Search this registry for the feature or domain keyword.
2. Open the owning paths listed here.
3. Check `docs/contributing/roadmap.md` for public status context.
4. Extend the existing surface instead of creating a parallel copy.

Status legend: `complete` means the main implementation exists; `active` means
recently implemented or being refined; `planned` means a public direction exists
but code should be checked first; `blocked` means do not start without an
explicit maintainer decision.
