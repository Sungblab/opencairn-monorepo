  • → 신규 사용자는 헤더로 "어디 있는지" 즉시 인지
  • → 파워 사용자는 레일로 1클릭 전환 (탭 여러 프로젝트 혼용 가능)
  • → 프로젝트 설정/공유/삭제는 헤더 드롭다운에 둠 (레일은 전환 전용)
  • → 탭 여러 개 열어도 탭별로 다른 프로젝트도 가능 (레일이 탭별 상태 따라감)
조합안 — 격리감 + 전환 편의성 + 브랜딩 세 마리

"← 프로젝트 목록으로" 클릭 시
메인 영역이 프로젝트 선택 오버레이로 전환됨. 전환 마찰은 의도된 것 — "다른 프로젝트에 간다"를 사용자가 명시적으로 선언.
  • 투자노트 (현재)
  • 리서치 노트 — 12개 문서
  • 독서 메모 — 15개 문서

Pros

  • 격리감 극대치 — 다른 프로젝트의 존재조차 전환 전엔 안 보임
  • 한 프로젝트에 "몰입"하는 장기 작업에 좋음
  • 사이드바 최소주의 (UI 경량)

Cons

  • 다른 프로젝트 간 빠른 점프 불가 — 전환 마찰 큼
  • 탭을 여러 프로젝트에 걸쳐서 쓰기 어려움 (프로젝트 바꾸면 탭 초기화 느낌)
  • AI 어시스턴트에 "다른 프로젝트 문서 참조" 요청도 까다로워짐
참고: 위 4개는 격리 표현 방식의 축이지, 채택된 안은 세부 요소(워크스페이스 스위처 위치 · 검색바 위치 · 유저 푸터 등)를 더 다듬음. 사이드바 이외 영역(탭바 · 채팅 패널)은 다음 단계에서 붙여. 각 옵션의 사이드바 폭은 1=48+232px / 2=280px / 3=264px / 4=264px.
