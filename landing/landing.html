    Attention Mechanism

    compiled by 3 agents · 12 sources · updated 2h ago

    Attention은 입력 시퀀스의 각 토큰에 가중치를 부여해 관련성을 계산하는 메커니즘입니다. Vaswani et al. (2017)의 "Attention Is All You Need"가 transformer 아키텍처의 기반을 제시했으며, 이후 self-attention과 multi-head attention으로 확장되었습니다.

    ⚠ CONTRADICTION
    자료 3개가 quadratic complexity에 대해 다른 수치를 제시합니다. Linear Attention 계열은 이 가정을 건드립니다.

    관련 문서: Transformer, Scaled Dot-Product, Linear Attention.

    [ 008 / 012 · PRINCIPLES ]

    네 가지 약속이
    한 자리에 모이기는 드뭅니다.

    자료 주권 · 살아 있는 지식 · 학습까지 한 줄기 · 열린 소스. 한두 가지를 지키는 도구는 많지만, 넷을 동시에 두는 자리가 OpenCairn입니다.

    [ a ] sovereignty

    자료는 내 서버에 머뭅니다.

    텍스트 · 벡터 · 원본 파일 모두 여러분의 인프라 안에 있습니다. 완전 로컬 LLM 모드에서는 외부로 나가는 요청이 0. 외부 SaaS를 쓸 수 없는 환경에도 그대로 도입 가능합니다.

    [ b ] compile

    정리는 AI, 판단은 사람.

    Compiler 에이전트가 개념을 뽑아 위키 페이지를 만들고, 기존 위키와 연결합니다. 여러분은 결과를 검토하고 편집하기만 하면 됩니다. 손으로 옮겨 적는 시간이 사라집니다.

    [ c ] one·throughline

    저장 → 탐색 → 학습까지 한 줄기.

    Q&A · 5뷰 그래프 · 간격 반복 플래시카드 · 소크라테스식 문답 · 오디오 팟캐스트까지 같은 위키를 재료로 돌아갑니다. 도구 사이를 건너뛰지 않습니다.

    [ d ] open·forever

    AGPLv3로 공개, 나가는 길 항상 열림.

    원하면 포크하고 고치고 직접 배포할 수 있습니다. 사내 사용에는 제약이 없으며, 상용 계약이 필요한 경우 별도 라이선스도 준비되어 있습니다. 서비스가 끊길 걱정이 없습니다.

    한 번의 명령어로
    여러분의 서버에서.

    • →월 1회 자동 백업 복구 리허설 스크립트 포함
    • →Cloudflare R2 / MinIO / S3 중 선택
    TERMINAL
    $ git clone https://github.com/Sungblab/opencairn-monorepo.git
    $ cd opencairn-monorepo
    $ cp .env.example .env
    # .env에 LLM_API_KEY 설정 (Gemini) 또는 LLM_PROVIDER=ollama
    $ docker-compose up -d
    # Ollama 완전 로컬:
    $ docker-compose --profile ollama up -d
    → http://localhost:3000

    전부 여러분이 운영하는 인프라에 저장됩니다. 텍스트·벡터·위키는 데이터베이스에, 원본 파일은 파일 스토리지에. Gemini 모드에서는 AI 호출만 외부로 나가고, 완전 로컬 모드에서는 외부 요청이 0입니다.

    AGPLv3면 회사에서 못 쓰는 거 아닌가요? +

    사내 사용은 전혀 문제없습니다. AGPLv3는 "네트워크를 통해 외부 제3자에게 서비스할 때" 소스 공개 의무가 생깁니다. 사내 직원 간 공유는 배포가 아니라 사용으로 간주됩니다. 소스 공개가 곤란한 경우 상용 라이선스로 전환 가능.

    Gemini 말고 OpenAI는 안 되나요? +

    v0.1에서는 Gemini(호스티드 · BYOK)와 Ollama(완전 로컬) 두 가지를 지원합니다. 대용량 컨텍스트 캐싱 · 네이티브 오디오 생성 · 실시간 웹 검색 같은 Gemini 고유 기능을 핵심 가치로 삼기 때문입니다. 완전 로컬이 필요한 경우 Ollama.

    기존 노트 앱에서 마이그레이션할 수 있나요? +

    Markdown 형태로 내보낸 파일을 업로드하면 Compiler가 위키로 다시 엮습니다. 손으로 정리하던 기존 페이지들이 연결된 위키로 되살아납니다. Notion · Obsidian · Bear 등 주요 앱의 내보내기 포맷을 지원합니다.

    AI가 위키를 잘못 쓰면 어떻게 되나요? +

    AI가 생성한 문장에는 자동 표시가 붙어 언제든 직접 수정할 수 있습니다. 한 번 손으로 편집한 부분은 이후 AI가 덮어쓰지 않습니다. Librarian이 모순을 찾아 경고하고, 모든 변경은 활동 로그에 남아 원하는 시점으로 되돌릴 수 있습니다.

    실시간 협업은 얼마나 매끄러운가요? +

    Hocuspocus 기반의 현대적 동기화를 사용해 충돌 해결이나 "다시 연결 중" 같은 지연이 거의 없습니다. 커서·아바타 표시, 블록 단위 코멘트, @멘션, 알림, 공개 링크, 게스트 초대까지 v0.1부터 제공됩니다.
